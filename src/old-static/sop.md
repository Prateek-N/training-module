📋 INTERVIEW PROXY SOP CHECKLIST

⚠️ Accountability Warning

Any deviation from this checklist—especially missing MANDATORY items—makes the expert fully responsible if a complaint arises.

📄 RESUME UNDERSTANDING – SOP

✅ Mandatory (Before Session)

🔧 Create end-to-end project document based on resume

☎️ Conduct prep call with candidate to assess:

What they know about their projects

Familiarity with tech stack

📚 Draft detailed project breakdown (architecture, role, tech stack)

🗓️ Confirm session date & time

📝 Prepare project storyline (for interview usage)

⚪ Optional (Before Session)

👥 Schedule mock interview if it hasn't been done before

🔍 Check if candidate can articulate business logic naturally

✅ Mandatory (During Session)

🧠 Explain each resume point with real-world context

🗂️ Share Google Doc:

Resume details

Tech stack definitions (layman-style)

Support system how-to (Teams, audio share, screen share)

Tips on handling delays in answering naturally

🎤 Test audio share by playing sample audio

✅ Mandatory (After Session)

📤 Share final document with the candidate

🧾 Explain support process clearly:

How we connect (Teams meeting)

What to expect during real-time support

Audio/screen expectations

✅ Confirm candidate comfort with the storyline & environment

📌 Tag task complete with timestamp + your initials

🔍 SCREENING / FIRST INTERVIEW SUPPORT – SOP

✅ Mandatory (Before Support)

📥 Acknowledge request IMMEDIATELY via email

📅 Create calendar invite immediately (even for future dates)

👥 CC Manager + TL (Marketing) in invite

📞 Call candidate at:

30 minutes before

20 minutes before

10 minutes before

🎧 Test screen + audio sharing again

🗂️ Share support document with:

Prepared answers

Project storyline

Connective methodology for candidate familiarity

⚪ Optional (Before Support)

📄 Share "How to behave in interviews" cheat sheet

🧪 Schedule a last-minute mock if time allows

✅ Mandatory (During Support)

👂 Match candidate’s tone, don’t overfeed answers

📌 Add new Qs & responses into Google Doc live

🕰️ Ensure no long delays or unnatural pauses

🔄 Actively update document as round progresses

✅ Mandatory (After Support)

🧾 Give detailed feedback to candidate (no one-liners)

📧 Send structured feedback email:

What was asked

What was answered

What went wrong/right

What to fix

🏷️ Close ticket with complete notes in system

💻 TECHNICAL / FINAL ROUND SUPPORT – SOP

✅ Mandatory (Before Support)

🔁 Follow all pre-check steps from First Round

📂 Prepare tech Q&A with natural answers, no textbook definitions

🧠 Refresh candidate on project storyline

📉 Warn candidate not to give long-winded answers

✅ Mandatory (During Support)

💬 Guide answers naturally, example-based

💻 Obtain remote access and write answers/code in preferred language

👨‍💻 For Coding Questions:

Algorithm or logic explained FIRST

Use single-char vars (unless SQL)

Code is clean and direct

Line-by-line non-tech explanation with examples

Add time/space complexity

Include worst-case scenario

✅ Mandatory (After Support)

✍️ Send full breakdown email (no generic notes)

📝 Include:

What happened

What was expected

What was missed

Support feedback

🗃️ Log session, flag for improvement if needed

🧪 ASSESSMENT SUPPORT – SOP

✅ Mandatory (Before Assessment)

🔁 Follow Interview Support setup (Teams, Google Doc, recording)

🧠 Understand platform restrictions (screen share, proctoring, etc.)

🧾 Confirm candidate’s preferred programming language

✅ Mandatory (During Assessment)

💻 Obtain remote access and write answers/code in preferred language

❌ If access not granted:

Expert must still attend, observe, and provide support via Google Doc

Comment on syntax errors in real time

Email confirmation that access was denied

Only if all above are fulfilled may expert hold candidate accountable

📊 Complete all technical, logical, and aptitude-based questions:

System design (e.g., Amazon)

English grammar/mathematical reasoning

🎥 Recording is MANDATORY. No recording = expert is at fault by default

✅ Mandatory (After Assessment)

🧾 Share complete feedback:

Code/logic review

How candidate handled prompts

What was completed vs. skipped

📧 Send assessment summary with attached Google Doc

🗂️ Mark task status + comments in email thread

🗓️ Create meeting invite regardless of date/time
